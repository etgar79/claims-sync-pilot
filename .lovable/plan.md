## מה משתנה במסכי ההקלטות

כיום הפאנל המורחב עם הנגן, עריכת התמלול ושינוי שמות דוברים קיים רק במסך **תמלולים**. בכרטיסי ההקלטות עצמם (הקלטות שטח / הקלטות פגישה / מסך הפגישות) עדיין נפתח דיאלוג נפרד. נחבר את אותו פאנל ישירות לכרטיס, ונוסיף יכולות שחסרו: בחירה בין גרסאות תמלול, מיזוג ביניהן, ועריכת שם פגישה ישירות מהממשק.

## מה תקבלי בכל כרטיס הקלטה

לוחצים על "הצג / ערוך" בכרטיס — והכרטיס מתרחב מתחתיו לפאנל עבודה מלא:

- **נגן אודיו מובנה** — להאזין תוך כדי קריאה.
- **עריכת תמלול בתוך הכרטיס** עם שמירה אוטומטית.
- **שינוי שמות דוברים גלובלי** — מזהים אוטומטית את הדוברים, ממלאים שם → "החל על כל התמלול" משנה בכל מקום ושומר.
- **בחירת גרסת תמלול** — לכל הקלטה נשמרות מספר גרסאות (תמלול-על מפיק כמה במקביל). מוסיפים כפתור **"השתמש כגרסה הראשית"** ליד כל גרסה בהיסטוריה — לחיצה תקבע אותה כתמלול הפעיל של ההקלטה.
- **מיזוג גרסאות** — כפתור **"מזג גרסאות"** פותח דיאלוג בחירה (מבוסס `MergeTranscriptsDialog` הקיים) שמייצר תמלול אחד משולב באיכות גבוהה ושומר אותו כראשי.
- **ייצוא ושיתוף** — PDF / TXT / העתקה / פתיחה ב-Drive — מתוך אותו פאנל.

## מה נוסף לפגישות

- **עריכת שם פגישה (וגם שדות בסיסיים) ישירות מהממשק**: בכותרת `/meetings/:id` יתווסף כפתור עיפרון שיפתח דיאלוג קצר (כותרת, לקוח, פרויקט, מיקום, תאריך) עם שמירה ל-`meetings`. גם בכרטיס הפגישה ברשימה `/meetings` יתווסף כפתור "ערוך" קטן.
- בנוסף, באותו פאנל המורחב של הקלטה — אם ההקלטה משויכת לפגישה — יופיע ליד שם הפגישה כפתור עיפרון לעדכון שם הפגישה במקום, בלי לעזוב את המסך.

## פירוט טכני

### 1. הרחבת `ExpandableTranscriptPanel.tsx`
- להוסיף כפתור **"השתמש כגרסה הזאת"** בלולאת `versions.map(...)` — קורא ל-`update` של `recordings/meeting_recordings` עם `transcript = version.transcript` ו-`transcription_service = version.service`, ואז `setEdited` + `onUpdated`.
- להוסיף כפתור **"מזג גרסאות"** שפותח את `MergeTranscriptsDialog` הקיים, ומעביר אליו `recordingId`, `table` ואת `versions`. בסיום — `loadVersions()` + `onUpdated()`.
- להוסיף שורה אופציונלית "פגישה: X ✎" כשמועבר `meetingId` + שם, עם דיאלוג עריכה קצר שמעדכן `meetings.title`.

### 2. שילוב הפאנל ב-`RecordingCard.tsx`
- להוסיף prop חדש `expandedSlot?: ReactNode` שיירונדר מתחת לאזור הפעולות (בתוך אותו `Card`), כדי שהכרטיס עצמו "יגדל" כשפותחים.
- להוסיף prop `expanded: boolean` + `onToggleExpand` — כפתורי "צפה/ערוך" יחליפו `setViewTarget` בהפעלת `onToggleExpand("view"|"edit")`.

### 3. עדכון `Recordings.tsx` ו-`MeetingRecordings.tsx`
- להוסיף state `expandedId` + `expandedMode`.
- ב-`renderCard`: להעביר `expanded`, `onToggleExpand`, ו-`expandedSlot={<ExpandableTranscriptPanel ... />}`.
- להעביר `meetingTitle` / `caseTitle` ל-`item.context`, ו-`onAssign` להפעיל את אותם דיאלוגי השיוך הקיימים.
- להסיר את `TranscriptViewerDialog` מהזרימה הראשית (להשאיר רק כ"חלון מלא" אופציונלי).

### 4. עריכת שם פגישה
- ב-`Meetings.tsx`: כפתור עיפרון בכרטיס שפותח את אותו `Dialog` הקיים של "צור פגישה" במצב עריכה (prefill + UPDATE במקום INSERT).
- ב-`MeetingDetail.tsx`: כפתור עיפרון ליד הכותרת שפותח את אותו דיאלוג.

### 5. אבטחה
RLS הקיים (Users update own / Admins manage all) כבר מכסה את כל ה-UPDATEs המתוארים — אין שינויי סכימה או policies נדרשים.
