
# סדר מלא במערכת — ליבת תמלול + אדריכל + שמאי + אדמין

המערכת היא במהותה **מערכת תמלול**. עליה נבנים שני workspace-ים שמשתמשים באותה ליבה:
- **שמאי**: תיקים, הקלטות שטח, דוחות שמאות
- **אדריכל**: פגישות, הקלטות פגישה, סיכומי פגישה
- **אדמין**: מנהל את ה-API, רואה צריכה, יוצר משתמשים — **לא מנהל תיקים/פגישות בעצמו**

כל יוזר (כולל אדמין) מקבל:
1. תיקיית Drive להקלטות שטח (אם הוא שמאי)
2. תיקיית Drive להקלטות פגישות (אם הוא אדריכל)
3. כפתור "תמלל" על כל הקלטה — תמלול-על או תמלול מהיר

## הבעיה הנוכחית (מה ראיתי בקוד+DB)

1. ליוזר `9aa018ac` (שלומי?) הוקצו **שלושת התפקידים** ידנית בעבר → לכן הוא רואה הכל. זה דאטה ולא קוד.
2. `useUserRoles` מחזיר `isAppraiser=true` ו-`isArchitect=true` לכל אדמין → גורם לעמוד ההגדרות להציג שתי תיקיות במקום רק את הרלוונטיות לתפקידיו האמיתיים.
3. `Index.tsx` (דשבורד שמאי) רץ `seedSampleCases` כשיש תפקיד `appraiser` — בסדר, אבל היוזר הזה הוא גם appraiser ידנית, אז זרעו לו.
4. אין UI נפרד שאומר לאדמין "אתה לא צריך תיקיית הקלטות אישית — אלא אם הקצית לעצמך תפקיד עבודה".
5. `seedSampleCases` עוד שותל נתוני דוגמה — מבלבל ויוצר רושם שהאדמין הוא שמאי.

## מה נעשה

### 1. ניקוי דאטה (קודם כל)
- להסיר מהיוזר 9aa018ac את התפקידים `appraiser` ו-`architect` — שיישאר admin בלבד.
- למחוק את תיקי הדוגמה (`cases` עם `case_number` מהסט הזרוע) שלו אם נוצרו.
- לוודא שיוזר d5baeff5 הוא רק architect ו-f8227de7 רק appraiser (כבר ככה).

### 2. הפרדת תפקידים נכונה (קוד)
- `useUserRoles`: להחזיר `isAppraiser`/`isArchitect` **רק** לפי הקצאה מפורשת — בלי `|| isAdmin`. אדמין יקבל גישה לדפי עבודה דרך `ProtectedRoute` בלבד אם הוא בחר workspace מתאים, אבל לא ייחשב אוטומטית כשמאי/אדריכל בעמודי הגדרות וזרעי דוגמה.
- `Index.tsx`: להסיר לחלוטין את `seedSampleCases` (זה מקור בלבול). דשבורד שמאי ריק = ריק.
- `ProtectedRoute`: אדמין נכנס לכל route רק אם בחר את ה-workspace המתאים (`useActiveWorkspace`), לא אוטומטית. כך כשאדמין בוחר "ניהול מערכת" הוא לא יכול בטעות לפתוח `/cases` ולראות תיקים.

### 3. ליבת תמלול אחידה לכל יוזר
זה **כבר קיים** ועובד עבור שני הסוגים של הקלטות (`useTranscribeAll` עובד גם על `recordings` וגם על `meeting_recordings`). מה שחסר:
- **דף "התמלולים שלי"** משותף לכל יוזר (כולל אדמין) — מציג את כל ההקלטות שלו (משתי הטבלאות) עם סטטוס תמלול וכפתור "תמלל".
- כפתור "תמלל" כבר קיים בשני המסכים. נוודא שהוא עקבי.

### 4. ה-Settings — לפי התפקיד האמיתי
- **תיקיית הקלטות שטח** מוצגת רק אם יש לו `appraiser` מפורש.
- **תיקיית הקלטות פגישות** מוצגת רק אם יש לו `architect` מפורש.
- אדמין שלא הקצה לעצמו תפקיד עבודה → לא יראה אף בורר תיקייה (לא צריך).
- **הגדרות AI** ו**גיבוי** — admin only (כבר ככה).

### 5. תפריט צד נקי
- אדמין רואה: סקירה כללית, ניהול משתמשים, צריכה, הגדרות. אין לו תיקים/פגישות בתפריט אלא אם הקצה לעצמו תפקיד עבודה ובחר אותו ב-WorkspaceSwitcher.
- אדריכל רואה: דשבורד פגישות, פגישות, הקלטות פגישה, לקוחות, תבניות, הגדרות.
- שמאי רואה: דשבורד שמאי, תיקים, לקוחות, הקלטות שטח, תבניות, הגדרות.
- שני התפריטים מסתיימים ב"הגדרות" שמובילה לאותו עמוד עם הסעיפים הרלוונטיים בלבד.

### 6. מסך RoleHome
- ללא תפקיד → "לא הוגדר תפקיד" + יציאה (כבר קיים).
- אדמין → סקירת מערכת (כבר קיים).
- שמאי → דשבורד שמאי (Index ללא ה-seed).
- אדריכל → דשבורד פגישות (כבר קיים).
- אדמין שגם הקצה לעצמו תפקיד עבודה → ברירת מחדל admin, יכול לעבור ב-WorkspaceSwitcher.

## פירוט טכני

**קבצים שיתעדכנו:**
- `src/hooks/useUserRoles.ts` — `isAppraiser`/`isArchitect` בלי fallback ל-admin.
- `src/components/ProtectedRoute.tsx` — אדמין מקבל גישה רק אם `workspace` תואם (או admin routes).
- `src/pages/Index.tsx` — להסיר את `seedSampleCases` והייבוא שלו.
- `src/lib/seedSampleCases.ts` — אפשר למחוק (לא יהיה בשימוש).
- `src/pages/Settings.tsx` — הסעיפים תלויים ב-roles האמיתיים בלבד (כבר חצי-ככה, נוודא).
- `src/components/AppSidebar.tsx` — אם stillLoading או אין workspace, להציג רק את "הגדרות" כדי שלא יהיה מסך ריק.

**מיגרציות SQL:**
```sql
-- הסר תפקידי עבודה משלומי (אדמין בלבד)
DELETE FROM public.user_roles
WHERE user_id = '9aa018ac-b2e5-45b0-a704-a493e77e206b'
  AND role IN ('appraiser','architect');

-- מחק תיקי דוגמה שזרעו לאדמין
DELETE FROM public.cases
WHERE user_id = '9aa018ac-b2e5-45b0-a704-a493e77e206b';
```

**מה לא משתנה:**
- סכמת DB, RLS, edge functions, הליבה של תמלול, חיבור Drive — כולם בסדר.
- כל יוזר מסנכרן לתיקייה האישית שלו ב-Drive (כבר ככה דרך `drive_work_folders` עם `user_id`).
- אדמין מנהל את מפתחות ה-AI (כבר ככה — הסעיף ב-Settings מוגן ב-`isAdmin`).

## תוצאה צפויה

- **שלומי (אדמין)** → נכנס ורואה רק "סקירה כללית / משתמשים / צריכה / הגדרות". בלי תיקים, בלי פגישות, בלי תיקיות הקלטות.
- **משתמש אדריכל** → רואה רק את עולם הפגישות + תיקיית הקלטות פגישה ב-Drive + תמלול.
- **משתמש שמאי** → רואה רק את עולם השומה + תיקיית הקלטות שטח ב-Drive + תמלול.
- **כל אחד** מקבל את אותה ליבת תמלול עם כפתור "תמלל" (תמלול-על / תמלול מהיר).
