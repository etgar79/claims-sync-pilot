
## מטרה
להפוך את המערכת לפשוטה, ידידותית ונגישה — כל המשתמשים יכולים לתמלל, ממשק עריכה אחד נקי, ו"תמלול-על" שמריץ את כל המנועים ברקע ו-Gemini ממזג אותם לתמלול אחד מאוחד.

---

## 1. תמלול-על במנגנון מאוחד (Gemini Merge)
היום `useTranscribeAll` מריץ 3 שירותים אבל המשתמש רואה גרסאות נפרדות בהיסטוריה. נשנה כך שלמשתמש תוצג **תוצאה אחת בלבד**:

- **edge function חדש**: `merge-transcripts`
  - קלט: `recording_id` + מערך גרסאות (טקסטים מ-3 המנועים).
  - שולח ל-Lovable AI Gateway (`google/gemini-2.5-pro`) prompt שמבקש למזג את שלוש הגרסאות לתמלול אחד מדויק בעברית, לתקן טעויות שכיחות, לזהות דוברים אחידים, ולשמור על נאמנות לשמע.
  - מחזיר טקסט סופי ומעדכן `recordings/meeting_recordings.transcript` עם `transcription_service: "merged"`.
- **`useTranscribeAll`** יקרא ל-`merge-transcripts` בסיום 3 ההפעלות (כבר שומר גרסאות ב-`transcript_versions`, צריך רק להוסיף את צעד המיזוג).
- בממשק: המשתמש רואה רק "תמלול" אחד. הגרסאות הגולמיות נשמרות ב-`transcript_versions` כגיבוי שניתן לראות בלשונית "היסטוריה" אם רוצים, אבל ללא שמות ספקים — תמיד דרך `serviceLabel()`.

## 2. עריכת תמלול — בלוק יחיד פשוט (`TranscriptViewerDialog.tsx`)
היום בלשונית "עריכה" יש שני בלוקים נפרדים (דוברים + מילות מפתח) עם 4 כפתורים. מאחדים:

- אזור טקסט גדול לעריכה ידנית.
- **בלוק יחיד "החלפות בטקסט"**:
  - שורות הדוברים שזוהו בראש (תג "דובר", "חפש" נעול, "החלף ל...").
  - מתחת — שורות `חפש ← החלף` חופשיות עם `+`/`×`.
  - **כפתור פעולה אחד**: "החלף בכל הטקסט ושמור".
- כפתורי "איפוס" ו"שמור" של העורך הידני נשארים.
- היסטוריית גרסאות → אקורדיון מתקפל סגור כברירת מחדל.

## 3. ביטול תמלול אוטומטי — תמיד לפי בקשה
- וידוא שאין הפעלה אוטומטית של תמלול אחרי `drive-sync` או העלאה — רק `transcript_status: "pending"`.
- כפתור "תמלל" תמיד פותח את דיאלוג הבחירה (תמלול מהיר / תמלול-על מאוחד).
- מסירים כל קיצור-דרך שמתחיל תמלול בלי בחירה מפורשת.

## 4. נגישות לכל המשתמשים
- כפתור "תמלל", פתיחת `TranscriptViewerDialog`, ועריכה — זמינים לכל משתמש מחובר בכל workspace (אדריכל / שמאי / אדמין). מסירים בדיקות תפקיד שמסתירות פעולות אלה.
- קישור "תמלולים" בסיידבר — לכולם.
- כפתור "ערוך תמלול" זמין מכל מקום שמציג תמלול.

## 5. שיפורי UX שמייצרים תחושת "כיף ופשטות"
- **Empty states ידידותיים**: אייקון גדול + משפט מנחה + כפתור פעולה ראשי (למשל בעמוד הקלטות ריק: "עוד אין הקלטות — סנכרן מ-Drive או הקלט עכשיו" + 2 כפתורים).
- **תגיות סטטוס צבעוניות**: ממתין (אפור), מתמלל (כחול עם אנימציה), הושלם (ירוק עם ✓), שגיאה (אדום).
- **Toasts קצרים**: "נשמר ✓", "מתחיל לתמלל...", "התמלול מוכן 🎉".
- **Tooltips** על כל אייקון-כפתור.
- **כותרות עמודים אחידות**: אייקון + שם + תיאור משנה קצר.
- אנימציית "טוחנים את 3 המנועים → ממזגים → מוכן" בזמן תמלול-על כדי שהמשתמש מבין שזה תהליך אחד שלם.

## 6. מה לא משתנה
- מבנה ה-Drive, RLS, בידוד תפקידים.
- שמות ספקי תמלול לא נחשפים — `serviceLabel()` תמיד.
