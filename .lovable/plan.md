
# פיצול אוטומטי לקבצים גדולים + תמלול ואיחוד

## הבעיה

קבצים גדולים מ-25MB (כמו "לילך וויס" 62MB) נכשלים:
- Whisper מוגבל ל-25MB
- Lovable AI/Gemini מוגבל ל-20MB
- ElevenLabs דורש מנוי בתשלום (החשבון החינמי שלך נחסם)

המנועים הזולים/חינמיים תומכים רק בקבצים קטנים. הפתרון: **לפצל בצד הלקוח** את ההקלטה לקטעים של ~15MB, לתמלל כל קטע במנוע הזול, ולאחד את התמלולים לפי הסדר.

## מה ייבנה

### 1. פיצול אוטומטי בדפדפן

מודול חדש `src/lib/audioSplitter.ts` שמשתמש ב-Web Audio API:
- מקבל קובץ אודיו (File/Blob)
- מפענח אותו ל-AudioBuffer
- מחלק לקטעים של ~10 דקות (≈15MB ב-MP3 128kbps)
- מקודד כל קטע ל-WAV (פורמט שכל המנועים מקבלים) או דוחס ל-MP3 דרך `lamejs`
- מחזיר מערך של Blobs מסודרים לפי זמן

הפיצול נעשה על **גבולות שקט** קצרים (חיפוש 3 שניות אחורה מנקודת החיתוך כדי למצוא את הנקודה הכי שקטה) — מונע חיתוך באמצע מילה.

### 2. תמלול מקבילי + איחוד

עדכון `src/components/TranscribeDialog.tsx`:
- אם הקובץ < 20MB → התנהגות נוכחית (קריאה אחת ל-`transcribe-audio`)
- אם הקובץ ≥ 20MB → פיצול ואז:
  - שליחת כל קטע במקביל (עם הגבלת concurrency של 2) ל-`transcribe-audio`
  - מעקב התקדמות: "מתמלל קטע 3 מתוך 7..."
  - איחוד התמלולים לפי סדר עם הפרדת `\n\n` בין קטעים
  - שמירה ל-`recordings.transcript` כמו רגיל

### 3. UI התקדמות

ב-TranscribeDialog יתווסף:
- בר התקדמות (`Progress`) שמציג כמה קטעים תומללו
- טקסט: "קובץ גדול — מפוצל אוטומטית ל-N חלקים"
- אם קטע אחד נכשל — נסיון חוזר אוטומטי (עד 2 פעמים), ואם עדיין נכשל — שמירת התמלול החלקי + סימון `[קטע X לא תומלל]`

### 4. רישום שימוש

כל קטע נרשם בנפרד ב-`usage_events` (כפי שקורה היום), כך שהמשתמש רואה את העלות הכוללת בעמוד Usage.

## קבצים שיתעדכנו

- `src/lib/audioSplitter.ts` (חדש) — לוגיקת פיצול
- `src/components/TranscribeDialog.tsx` — שילוב הפיצול + UI התקדמות
- `src/hooks/useTranscribeAll.ts` — תמיכה בפיצול גם בתמלול-על מרובה הקלטות
- `package.json` — הוספת `lamejs` לדחיסת MP3 (אופציונלי; אפשר גם בלי, רק WAV)

## הערות חשובות

- **אין שינוי במסד הנתונים** — שדה `transcript` הקיים מקבל את הטקסט המאוחד.
- **אין צורך ב-edge function חדש** — `transcribe-audio` הקיים נקרא פעמים רבות.
- הפיצול רץ בצד הלקוח, אז עומס השרת לא משתנה. החיסרון היחיד: על הטלפון/דפדפן ייקח זמן לפענח קובץ של 60MB (כדקה).
- מנועים שכבר תומכים בקבצים גדולים (כשתשדרגי את ElevenLabs בעתיד) ימשיכו לעבוד בלי פיצול — הלוגיקה רץ רק כשצריך.

## טיפול בקצוות

- קבצי `m4a/aac` שלא נטמעים נטיבית בכל הדפדפנים → fallback ל-`AudioContext.decodeAudioData` (תומך ב-Safari/Chrome/Edge העדכניים).
- אם הפענוח נכשל (פורמט נדיר) → הודעת שגיאה ברורה: "לא ניתן לפצל את הקובץ. אנא המירי ל-MP3."
